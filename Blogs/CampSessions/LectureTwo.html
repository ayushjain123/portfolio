Hosting Our First Website

So, Before we start hosting our website on "WEB" lets first understand what "WEB" is, how even if we host our web site we'll be able to fetch it, how does browser gets us our desired content how?????

Have you ever heard about that chacha in family who just tries to fix everyone's marriage!! . How is it related to our question?

It is related to our question, the way he gets rishta's for a person, this is exactly how browser works it gets to the server asks him Hey Do You Have Any Rishta Which Matches My Nephew's Requirement? Here `nephew` is client `rishta` is request which we made through browser(chacha), if server replies with yes it picks up all the content which we asked and puts it in our browser(client).

Now question arises, what is a server then?

A server provides data to different computers(CLIENT) and also, Server Sometimes act as mediators also. Its not necessary that server are BIG or have much storage capacity your PC can also be converted into a server.

In the process of deploying/hosting our website we understood how WEB works. Now comes another step to store the application which we create or to manage versions of our application

To manage these things we use GIT This application lets us store our application, track changes, view change history, invite different collaborators and deploy

lets Start with creating account on GIT

after completing all these steps you are ready to create a repository

What repository is now?

Repository is a folder that contains all the files of your application. Repositories can be public or private, public repositories are visible and can be cloned by others and private repository can be seen by a collaborator or owner

Now Creating Repository

README.md file is a markdown file which acts as html but is easily understandable by git and git converts text inside md file into a proper texts

gitignore is used to ignore some confidential files while pushing our code

After creating repository you will see thi type of interface.

Lets now understand how git works before getting into all these commands

GIT is a storage like Google Drive as we add our photos to google drive after creating account, similarly we add our file stored in our system locally and upload it on GIT

so now blurry picture of these commands will get a bit clear. By using these commands we tell our mediator GIT application to track all the changes we are making in a file while we code and after we are done coding we push our code to github

lets understand these commands now

git init This command allows git application to track that repository and initialises it with .git like any othe extension

git remote add origin ["GIT_URL"] By this command we tell our GIT application where to push all the code

You dont need to put commands mentioned above every time, these commands are entered when a repository is to be initialized

git add . this command adds all the files present in folder to GIT application and then git application keeps track of every single change until we commit all these changes

git commit -m "[COMMIT-MESSAGE]" this command finally saves every file. While commiting changes it is necessary to provide a commit message to track changes easily.

NOTE: ALL THESE COMMANDS ARE FOR GIT TERMINAL

Writing a good commit message itself is an art

These are some commit message notations

Finally after saving all the changes only thing left is to push every thing on github using this command. git push origin master now two question arises what is `origin` & `master`. So origin is local branch which we created while adding repository link master is github branch in which particular changes are updated

A repository can have many branches. These branches are used to work on different versions as while working in a team work is divided, to collaborate all the work together or to bifurcate the work branches are used
